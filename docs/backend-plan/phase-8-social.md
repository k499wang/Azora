# Phase 8: Social

## Goal

Only after the private tracking loop is working, explore optional social features.

## Non-Launch Rule

Social is explicitly not part of V1.

## Possible Features

- friends
- private achievement sharing
- optional weekly consistency sharing
- small groups
- consistency-based challenges

## Safety Constraint

Avoid social mechanics that encourage unsafe max-hold competition.

Do not build:

- public max-hold leaderboards
- challenge mechanics that pressure users to hold longer than they should

## Possible Data Model Later

- friendships
- shared achievements
- privacy/visibility settings

These stay deferred until the product direction is clear.

## Open Decisions

- friends-only vs groups vs public profiles
- what can be shared by default
- whether social is motivational enough to justify complexity

## Exit Criteria

- Any social layer is optional, privacy-aware, and does not distort the core safety model.
